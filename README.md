## 讲什么

| 页 | 要点 |
| --- | --- |
| 01–02 | 为什么升级：内容已就位，看现有沉淀规模 |
| 03 | 核心问题：缺的不是内容，是「调用层」 |
| 04–05 | 什么是 CLI：一份内容一次维护，人 / 程序 / AI 三处可调 |
| 06–07 | 三个调用口：**CLI 封装**、**MCP 接口**（AI 画稿/写码自动引用）、**网页问答**（design.html 门户一句话查规范） |
| 08–09 | 借鉴 OpenWiki 等成熟模式，先在核心 token / 组件规范上试点 `jdwiki` CLI |
