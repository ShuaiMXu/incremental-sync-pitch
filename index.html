什么是 CLI

一句话进、结果出的「命令行接口」

⌨️其实你早就在用

京东的 o2、飞书 CLI，都是 CLI——在终端敲一句，它给你结构化结果。

和图形界面（点按钮）最大的不同：CLI 能被程序和 AI 自动调，不用人守着点。

🔌为什么它是「调用层」的底座

Relay 的能力，已经被 zero 用 MCP 包起来了——所以 AI 能一句话调设计稿数据。

同理，把 wiki 内容包成 jdwiki CLI + MCP，规范就能被一句话调出、被任何工具和 AI 串起来。

京东已有先例：o2-kb（知识库 CLI，17 个子命令）、飞书 CLI 是 CLI 一路，zero（Relay MCP）是 MCP 一路——我们不是发明，是照着成熟模式做。

调用层的底座 · 借鉴 OpenWiki

调出来的，必须是新鲜、可信的内容

机制 ①
按版本增量保鲜

每篇记住「照的是设计稿哪一版」，设计稿一变自动更新受影响的那部分——像手机 App 只下更新包。保证调出来的规范不过期。

机制 ②
机器区 / 人工区分开

脚本只改机器区，同事手写经验的人工区一个字不碰。内容敢补、越补越厚，调出来的东西才越来越全。

来源
OpenWiki（LangChain 开源）

1.1万+ 收藏的开源工具，思路正好治我们的痛。工具本身用不上（源头在 Zero），但这两个保鲜机制直接借鉴。
